## 🔑 关键发现

### 1. 推理侧可调参数（用于Phase 5增广配置）

| 参数名 | 范围 | 增广效果 | 优先级 |
|--------|------|--------|--------|
| `seed` | 0 ~ 2^32-1 | 完全不同的视频生成 | ⭐⭐⭐ **最重要** |
| `noise_aug_strength` | 0.0 ~ 1.0 | 视频风格和质感变化 | ⭐⭐⭐ **最重要** |
| `num_inference_steps` | 1~50 | 质量 vs 速度权衡 | ⭐⭐ |
| `guidance_scale` | 0.0~15.0 | 与参考视频相似度 | ⭐⭐ |
| `scheduler` | DDIM/Karras/... | 去噪曲线 | ⭐ |
| `sample_stride` | 1~8 | 时间采样粒度 | ⭐ |
| `frames_overlap` | 0~15 | tile之间的平滑度 | ⭐ |

### 2. Medium预设的配置

```yaml
seeds: 20 个           # 0-19
noise_strengths: 11 个 # [0.0, 0.1, 0.2, ..., 1.0]
num_inference_steps: 2 # [25, 50]
guidance_scales: 2     # [2.0, 5.0]
schedulers: 1 个       # ["DDIM"]

总变体数 = 20 × 11 × 2 × 2 × 1 = 880变体/词汇
```

### 3. 推理专属的关键发现："潜空间帧插值增广"

**维度#7 是推理侧的独特增广方式**：

- **参数**: `frames_overlap`（范围0-15）
- **工作原理**: 将长视频分割成重叠的tiles，在潜空间中进行插值融合
- **效果**: 避免tile边界的不连续，生成平滑的长视频
- **这就是"多帧序列化"的真实含义**

## 🎯 使用指南

### 对于Pipeline设计者
- 参考本文档的**官方增广维度表**
- 了解推理侧的7个可调参数
- 设计配置文件时参考**Medium预设**

### 对于编排器开发者
- Phase 5增广配置应包含这7个参数
- 建议默认使用Medium预设
- `seed` 和 `noise_aug_strength` 是最关键的两个参数

### 对于数据增广测试
- 验证所有10个增广维度是否正常工作
- 重点验证**维度#7（潜空间帧插值增广）**
- 对比不同参数组合下的生成效果
