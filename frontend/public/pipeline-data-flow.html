Every task gets a directory under SHARED_DATA_ROOT/<task_id>/ with subfolders phase_1/output/ through phase_8/output/. Each phase reads from previous phase outputs (by absolute path) and writes only into its own phase_N/output/.

Worker filename ≠ phase number. Some workers were renumbered during history but files weren't renamed. Trust tasks.py phase routing and the function docstrings, not the filename. Mismatches are flagged below as [misnamed].

Phase 2.3 — Preprocess

Worker: backend/workers/phase5_preprocess.py::preprocess_videos [misnamed]. Five sub-steps; each writes to a dedicated subdir.

Inputs

upstreamphase_2/output/videos/<name>.mp4 (any source fps)
external scriptsUniSignMimicTurbo/scripts/sentence/{extract_all_frames_seq,filter_duplicate_frames,generate_videos_from_frames}.py

Outputs (5 sub-steps)

StepPathFormat
0phase_2/output/preprocess/organized/<stem>/<stem>.mp4Source mp4 re-encoded to 25 fps (or symlink if already 25 fps)
1phase_2/output/preprocess/frames/<stem>/<frame_id>_<ref_id>.jpgEvery frame extracted as jpg (25 fps)
2phase_2/output/preprocess/dedup/<stem>/<frame_id>_<ref_id>.jpgCumulative max-quadrant-diff dedup (thr=6.5%, accumulated across kept-frame intervals) — max-pooled diff% over 4 cells of the 70%W × 80%H central ROI, amplifies localized finger / one-hand motion
3phase_2/output/preprocess/resized_576/<stem>/<frame_id>_<ref_id>.jpgShort-edge scale + center crop to 576×576
4phase_2/output/preprocess/videos/<stem>.mp4576×576 H.264 mp4 @ 25 fps — input to Phase 3

Frame jpg naming

<frame_id>_<ref_id>.jpg — frame_id is zero-based int, ref_id is a per-source-clip ID (comes from the multi-source extraction tool; in single-clip mode it's just the stem).

Example (sid 69, Tareq)

raw 47fps × 767 frames (16.32s)
  → preprocess/organized/sid069/sid069.mp4      (25fps × 408 frames, same 16.32s)
  → preprocess/frames/sid069/{0..407}_sid069.jpg
  → preprocess/dedup/sid069/{...}_sid069.jpg    (168 jpgs kept, 41% retained)
  → preprocess/resized_576/sid069/{...}_sid069.jpg
  → preprocess/videos/sid069.mp4                (576×576, 25fps × 168 = 6.72s)

Example (sid 69 full pipeline trace)

preprocess input         168 frames @ 25fps (6.72s, 576×576)
  ↓ MimicMotion (stride=1)
  ↓ 169 frames @ 15fps (raw mp4, 11.27s playback)
  ↓ extract to jpgs + trim_inactive (3 frames cut from head/tail)
  ↓ 166 frames @ 25fps (regen)
  ↓ tail_glitch                                       → 166 frames
  ↓ realesr (2× SR)                                   → 166 @ 1152×1152
  ↓ scp back                                          → phase_3/output/videos/sid069.mp4
                                                        (~6.64s, sign moves at near-original speed)
Speed factor: with stride=1 + dedup thr=6.5 (max-quadrant), output is roughly half the duration of the raw recording (≈ 25fps / 47fps × 1/keep_ratio ≈ 0.5×). Was 4-10× faster with the upstream stride=2 default before the 2026-05-10 fix.

Phase 6 — Data Augmentation

Worker: backend/workers/phase7_augment.py::run_phase6_augment [misnamed — file is phase7_, actual phase 6]. backend/workers/phase6_framer.py is the 3D-view sub-component invoked from inside.

Two orthogonal axes:

  • Input category (which video set is augmented): sentence · word · segment — filtered by env PHASE6_CATEGORIES
  • Augmentation method (how it's augmented): cv_aug · temporal_aug · 3d_views · identity

Output dir layout: phase_6/output/<category>/<method>/...

Inputs

upstream (sentence)phase_2/output/videos/<stem>.mp4 — original sentence videos
upstream (word/segment)phase_5/output/segment_videos/<stem>_<n>.mp4 — word-level clips from Phase 5
externalFramerTurbo/ — diffusion model (3d_views method)
externalcv_aug Python package (cv_aug + temporal_aug methods)
externalMimicMotion w/ alt reference image (identity method)
envPHASE6_CATEGORIES = comma-separated subset of sentence,word,segment; default = all. Phase 8 only consumes word + segment so sentence aug is usually skipped.

Outputs

PathFormat
phase_6/output/<category>/cv_aug/<aug_id>/<video>.mp4OpenCV color/contrast/brightness-perturbed videos
phase_6/output/<category>/temporal_aug/<aug_id>/<video>.mp4Speed jitter / frame dropout variants
phase_6/output/<category>/3d_views/<aug_id>/<video>.mp4Novel-view synthesis (camera angle)
